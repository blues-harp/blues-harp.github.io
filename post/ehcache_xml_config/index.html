XML Configuration

介绍

使用XML文件你可以在创建CacheManeger的时候进行配置，根据一个XSD文件。

<config>根元素

我们XML文件的根元素。在一个XML文件里，只能有一个 <config>元素定义一个CacheManager。然而，使用Ehcache3.0的时候你可能想用一个XML配置文件创建多种CacheManager实例。与JSR-107javax.cache.spi.CachingProvider不同，Ehcache并不保存CacheManager实例的注册信息。

<service>元素

目前并不提供内建的服务。<service>元素是一个扩展点，用于指明CacheManeger管理的服务。每个这种方式定义的服务和CacheManeger有着一样的生命周期–对于每个为CacheManager定义的服务，Service.start方法会在CacheManager.init执行的时候调用；Service.stop方法会在CacheManager.stop执行的时候调用。

然后这些Service就可以被这个CacheManeger管理的Cache实例调用了。就像JSR-107配置部分解释的那样，JSR-107使用XML文件配置的这些扩展点（还有Ehcache的模块结构）。

<default-serializers>元素

一个<default-serializers>元素表示配置在CacheManager层级的Serializers。它是一个serializer标签的集合，每个serializer需要配置一个type和类的全名。

<default-copiers>元素

一个<default-copiers>元素表示配置在CacheManager层级的Copiers。它是一个Copier标签的集合，每个Copier需要配置一个type和类的全名。

<persistence>元素

一个<persistence>元素表示一个Persistence，在创建一个PersistentCacheManager的时候会用到。它需要配置一个持久化数据到磁盘的 directory位置。

<cache>元素

一个<cache>元素表示一个Cache实例，这个实例会由CacheManager创建和管理。每一个<cache>需要一个<alias>属性，用于在运行时通过org.ehcache.CacheManager.getCache(String, Class<K>, Class<V>)方法获取相应的Cache<K, V>实例。可选的uses-template属性让你引用一个<cache-template>元素的name属性。如果想了解详细的配置可以参考下面 cache-template 部分。

可选的子元素： 1. <key-type>: 这个Cache<K, V>的key的类全名（FQCN）;默认是java.lang.Object 2. <value-type>：这个Cache<K, V>value的类全名（FQCN）;默认是java.lang.Object 3. <expiry>：控制过期类型和它的参数 4. <eviction-veto>: org.ehcache.config.EvictionVeto<K, V>的实现的类全名，默认是null 5. <eviction-prioritizer>: org.ehcache.config.EvictionPrioritizer<K, V>的实现的类全名，默认是null 6. <integration>： 为一个cache-throuth模式配置一个CacheLoaderWriter 7. <resources>：配置不同层的容量。如果只使用堆内存，你可以直接使用元素代替它。

<cache-template>元素

<cache-template>元素表示一个被命名的（通过设置必须配置的name属性）模板，可以被<cache>元素继承。一个<cache>元素通过<uses-template属性指定一个name引用一个模板<cache-template>，然后就可以继承<cache-template>的所有属性。<cache>也可以根据需要覆盖这些属性。一个 <cache-template>元素甚至可以包含和一个<cache>一样的所有元素。

XML程序化解析

如果你想通过JSR-107 API获取你的CacheManeger,当你调用javax.cache.spi.CachingProvider.getCacheManager(java.net.URI, java.lang.ClassLoader).方法时，会自动执行下面的代码

final URL myUrl = this.getClass().getResource("/my-config.xml"); （1）
XmlConfiguration xmlConfig = new XmlConfiguration(myUrl);  （2）
CacheManager myCacheManager = CacheManagerBuilder.newCacheManager(xmlConfig); （3）

（1）获取一个指向XML文件的URL
（2）传入这个XML文件的ULR实例化一个XmlConfiguration
（3）静态方法org.ehcache.CacheManagerBuilder.newCacheManager(org.ehcache.config.Configuration)可以让你使用XmlConfiguration中的配置创建一个CacheManager对象。

我们也可以使用XML文件中定义的<cache-template>生成一个CacheConfigurationBuilder对象。假如一个配置文件中包含下面的XML段：

<ehcache:cache-template name="example">
  <ehcache:capacity>120</ehcache:capacity>
</ehcache:cache-template>

我们可以使用下面的代码创建一个基于example的CacheConfigurationBuilder：

CacheConfigurationBuilder<String, Object> cacheBuilder = xmlConfig.newCacheConfigurationBuilderFromTemplate("example", String.class, Object.class);  （1）
cacheBuilder.capacityConstraint(100L);  （2）

（1）创建一个builder，继承容量限制最多可以存120个entry，指定key的类型是java.lang.string
（2）要想覆盖继承的这个属性很简单，只需要在创建CacheConfiguration之前设置下这个属性就可以覆盖。
